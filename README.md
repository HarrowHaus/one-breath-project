# The One Breath Project

*Working name. Tagline: **Silent & Preventable.*** An independent nonprofit carbon-monoxide (CO) safety, education, and advocacy hub. Not a government site. Not a manufacturer.

This repository is the working directory for building the site with Claude Code. It contains the full project context, the build plan, and the operating rules — the actual application code is generated by Claude Code as you work through the phases.

## If you're the human running this (no coding needed)

You drive by pasting instructions to Claude Code, one phase at a time.

1. Open Claude Code with this repository as the working folder (desktop or the mobile app on the GitHub repo).
2. Claude Code automatically reads **`CLAUDE.md`** — the standing rules, including the git workflow and the five non-negotiables.
3. Tell it: *"Read the docs, then start Phase 0."* It will read `/docs` and open `/handoff/phases/PHASE_00_setup.md`.
4. After each phase, check the **"Done when"** line in that phase file. If it's met, move to the next phase. If not, use the check-and-report loop in `/handoff/README.md`.

You do **not** type git commands. Claude Code hands every commit and push to a small Haiku sub-agent (`.claude/agents/git-committer.md`) that pushes straight to `main` after each successful phase — no pull requests.

## What's in here

- **`GETTING_STARTED.md`** — zero-to-first-deploy runbook for the non-coder. Start here.
- **`CLAUDE.md`** — auto-read rules for Claude Code (git workflow, non-negotiables, working rhythm).
- **`.env.example`** — the shape of the secrets file (real secrets never go in the repo).
- **`/docs`** — the full context Claude Code reads before building:
  - `00_MASTER_CONTEXT.md` — project, stack, voice, non-negotiables.
  - `01_VOICE_GUIDE.md` — the civic-journalist voice + canonical copy.
  - `02_DO_NOT_BUILD.md` — the dark-pattern exclusion list.
  - `03_HERO_DOSSIER.md` — the sourced hero story + opening copy.
  - `04_BEHAVIORAL_ARCHITECTURE.md` — why people ignore CO and how the design overcomes it.
  - `05_DATA_PLATFORM.md` — data sources, ingestion, the internal API.
  - `06_DESIGN_SPEC.md` — exact colors, type, spacing, identity.
  - `07_SITEMAP_AND_ROUTES.md` — every page's fixed URL.
  - `08_DATA_DICTIONARY.md` — every metric + real seed data.
  - `09_STYLE_SHEET.md` — copy mechanics for consistency.
  - `10_MEASUREMENT.md` — what to instrument and how to read it.
  - `11_CONTENT_CALENDAR.md` — the seasonal timeliness engine.
  - `12_LEGAL_CHECKLIST.md` — privacy, 501(c)(3), attribution.

- **`/content`** — every user-facing word, pre-written and final, placed verbatim (home, journeys, the full Learn article set, experiences, data, resources, act, about, metadata, system pages). `content/README.md` maps pages to files and defines the data tokens.
- **`/handoff`** — the build:
  - `README.md` — how to run the phases.
  - `/phases` — `PHASE_00` … `PHASE_10`, in order.
  - `LAUNCH_CHECKLIST.md` · `TROUBLESHOOTING.md` · `FACT_CHECK_LOG.md`.
- **`.claude/agents/git-committer.md`** — the Haiku commit/push sub-agent.

## Accounts you'll need (mostly free, set up as phases ask)

GitHub (this repo) · Cloudflare Pages (deploy from GitHub) · Neon or Supabase (Postgres + PostGIS, Phase 2) · a free AirNow API key (Phase 2, optional) · the USFA National Fire Department Registry CSV download (Phase 2/8) · a privacy-first analytics account (Phase 10). Domain name can wait until the nonprofit's real name is chosen.

## Launch gate

The whole site can be **built** before it goes **public**. Do not launch publicly until: every published figure is sourced and tagged, and the legal basics are in place (privacy policy, accessibility statement, and the "independent nonprofit, not a government site" disclaimer). The hero is a public news story used with attribution, so no family permission is required to publish; notifying the family and honoring any takedown request is an optional courtesy.
