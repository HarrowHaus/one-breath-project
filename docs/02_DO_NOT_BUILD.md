# 02 — Do Not Build (dark-pattern exclusion list)

These are prohibited regardless of short-term conversion lift. Each maps to an FTC dark-pattern category (create false belief / conceal information / obscure or manipulate / unauthorized action). If a technique's effectiveness depends on the visitor *not* understanding it, it belongs on this list.

- **No pre-ticked** consent, donation, or subscription boxes.
- **No forced continuity / roach-motel:** unsubscribe and opt-out must be as easy as opt-in.
- **No confirmshaming** ("No thanks, I don't care about my family's safety").
- **No fake urgency or scarcity** (fake countdowns, "3 people viewing"). Real seasonal timeliness is allowed; fabricated urgency is not.
- **No manufactured false beliefs:** no invented statistics, no implying certainty where you have a modeled estimate, no truncated or misleading chart axes.
- **No concealed information:** costs, funding sources, and any manufacturer relationship are disclosed up front.
- **No buried actions or disguised ads.** The manufacturer/product appears only at a clearly labeled "find protection" step — never as the site's identity or woven into editorial.
- **No dark-pattern consent banners** for analytics — clear accept/decline parity.
- **No fabricated real-time counters** (e.g. a live "deaths today" ticker). An honest, clearly-labeled *average pace* is acceptable; invented live death data is not.

**The test:** would this still work if the visitor could see exactly what we're doing and why? If yes, keep it. If it only works hidden, cut it.

**USWDS note:** USWDS carries visual signals of *government* sites (the `.gov` banner, the Identifier). Using those on a nonprofit site would itself "create a false belief." Adopt USWDS's accessibility, components, and calm register — but brand clearly as an independent nonprofit and never impersonate a government identifier.
